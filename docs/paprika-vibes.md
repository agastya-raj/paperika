# paperika visual notes

This branch adds a paprika-inspired visual layer to the repo.

The goal was not to turn the project into fan-art soup. The goal was to give it a stronger identity: vivid red, dreamlike collage energy, and just enough theatricality to make `paperika` feel like a real thing instead of another efficient little Python cave.

## What changed

- Added a repo banner built from the uploaded reference images
- Updated `README.md` so the repo opens with the banner and a slightly more cinematic tone
- Updated `docs/e2e-notes.md` so the docs visually match the new identity
- Kept the technical content grounded; no runtime behavior was changed

## Main asset

- `assets/paprika/paperika-banner.svg`

## Suggested next step

If you want to take it one step further on GitHub itself, use this banner or a crop from it as the repository social preview image in Settings. That is the closest GitHub gives you to a real repo icon without acting like it is 2007 and branding is optional.
